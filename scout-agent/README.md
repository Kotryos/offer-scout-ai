# Scout Agent

Kotlin/Spring Boot WebFlux service for evaluating job offers with Spring AI tools.

## What It Does

- Accepts `POST /offer/evaluation`.
- Lets the model decide whether to call tools.
- Uses Tavily for web search and Jina Reader for page fetching.
- Returns JSON:

```json
{
  "evaluation": "..."
}
```

## Requirements

- Java 21
- Gradle wrapper included
- `GROQ_API_KEY`
- `TAVILY_API_KEY`
- optional `JINA_API_KEY`

## Run Locally

For local development, copy the example local profile:

```bash
cp src/main/resources/application-local.example.yml src/main/resources/application-local.yml
```

Fill in your local API keys. `application-local.yml` is ignored by Git.

Windows:

```powershell
.\gradlew.bat bootRun --args='--spring.profiles.active=local'
```

Unix:

```bash
./gradlew bootRun --args='--spring.profiles.active=local'
```

In IntelliJ IDEA, set the active profile on the Spring Boot run configuration:

```text
Active profiles: local
```

or add this VM option:

```text
-Dspring.profiles.active=local
```

## Example Request

`offerText` can be either plain job-offer text or a URL.

Plain text:

```bash
curl -X POST http://localhost:8080/offer/evaluation \
  -H "Content-Type: application/json" \
  -H "X-Correlation-ID: local-test" \
  -d '{"offerText":"Senior Kotlin backend role, remote, B2B","profileContext":"Senior Kotlin/Spring developer looking for remote work"}'
```

URL:

```bash
curl -X POST http://localhost:8080/offer/evaluation \
  -H "Content-Type: application/json" \
  -H "X-Correlation-ID: local-url-test" \
  -d '{"offerText":"https://example.com/jobs/senior-kotlin-engineer","profileContext":"Senior Kotlin/Spring developer looking for remote work"}'
```

## Architecture Notes

- Controllers return `Mono` and do not block the WebFlux event loop.
- `EvaluationService` offloads the blocking Spring AI call to `boundedElastic`.
- `WebTool` is the blocking adapter required by Spring AI `@Tool`.
- `webintegration` remains reactive and returns `Mono<String>`.
- `X-Correlation-ID` is read or generated by a filter and included in logs.
